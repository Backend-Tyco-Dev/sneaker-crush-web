# Content audit checklist (editorial / CMS)

Use this for AdSense “low value content” remediation. Engineering cannot replace these steps.

## Articles (GraphQL `Article` / news pages)

- [ ] Each published article has **enough original body text** to stand alone (avoid headline + one paragraph syndication only).
- [ ] Prefer **unique angles** or clear sourcing where content is industry news.
- [ ] **Audit** existing URLs: improve thin posts or unpublish / `noindex` per your SEO policy.

## Releases

- [ ] Prefer a **real `description`** in the CMS for each important release page. Empty descriptions fall back to auto-generated copy in the web app and can read as thin at scale.
- [ ] For low-priority or duplicate listings, decide whether to **limit indexing** (product/SEO decision).

## Evergreen / pillar content

- [ ] Maintain a small set of **guides or reference pages** (beyond infinite scroll lists) so the site reads as a destination, not only an aggregator.

## Ongoing

- [ ] Set a **minimum publishing bar** (e.g. length, sourcing) for new content.
- [ ] Re-check quarterly or after major CMS imports.
